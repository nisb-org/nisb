# Contributing to NISB

Thank you for your interest in NISB.

NISB is currently developed and maintained by its original creator. Feedback, bug reports, documentation improvements, and small carefully reviewed contributions are welcome.

## Project Ownership

Submitting an issue, pull request, patch, document, translation, suggestion, design, configuration, or other contribution does not make you an owner, co-owner, shareholder, partner, investor, employee, revenue-share participant, or co-founder of NISB.

The original NISB project, brand, roadmap, release decisions, licensing decisions, and commercial decisions remain controlled by the project maintainer.

Formal cooperation, investment, employment, revenue sharing, partnership, acquisition, or special commercial arrangement requires a separate written agreement.

## Pull Requests

A pull request is a proposed change.

The maintainer may accept it, reject it, request changes, close it, rewrite it, or use it as a reference for a separate implementation.

Please do not submit large pull requests without opening an issue first.

## Contribution License

NISB uses a dual licensing model:

1. AGPLv3 for open-source use.
2. Commercial licensing for users or organizations that need different terms.

By submitting a contribution to NISB, you agree that your contribution may be used, modified, distributed, sublicensed, and relicensed by the NISB maintainer as part of NISB under both:

1. The AGPLv3 open-source version.
2. Commercial, proprietary, enterprise, hosted, or private license terms offered by the NISB maintainer.

If you do not agree with this, please do not submit contributions.

## Contributor License Agreement

For code contributions, the maintainer may require acceptance of `CLA.md`.

The maintainer may refuse to merge code contributions unless the contributor has agreed to the contribution license terms.

## What Contributions Are Welcome

Good contributions include:

- Clear bug reports with reproduction steps.
- Documentation fixes.
- Typo fixes.
- Installation notes.
- Translation fixes.
- Small UI fixes.
- Small isolated bug fixes.
- Test cases.
- Security reports through the private security process.

## Large Changes Require Discussion

Please open an issue before working on:

- Room runtime architecture.
- MCP provider architecture.
- Federation or remote capability changes.
- Authentication or authorization.
- Database or storage changes.
- Licensing-related changes.
- Major frontend rewrites.
- Major backend rewrites.
- New commercial or enterprise features.

Unsolicited large pull requests may be closed without review.

## No Obligation

Submitting a contribution does not create an obligation for the maintainer to:

- Merge it.
- Maintain it.
- Provide support for it.
- Accept future related changes.
- Share revenue.
- Grant governance rights.
- Offer employment.
- Offer partnership.
- Offer commercial rights.

## Security

Do not report security vulnerabilities in public issues.

Use the process described in `SECURITY.md`.

## Maintainer Capacity

NISB is currently maintained by an independent maintainer.

Please keep issues focused, reproducible, and respectful.

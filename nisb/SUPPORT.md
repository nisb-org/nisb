# Support

NISB is an open-source project maintained by an independent maintainer.

Please read this page before opening issues or requesting help.

---

## Community support

Community support is best-effort.

You may open GitHub issues for:

- Reproducible bugs.
- Documentation errors.
- Installation problems with clear logs.
- Small feature requests.
- Regression reports.
- Security-safe reproduction cases.

Please do not include secrets, API keys, bearer tokens, private documents, private paths, customer data, private Room histories, or production credentials in public issues.

---

## Not included in free support

Free community support does not include:

- Guaranteed response time.
- Private debugging sessions.

- Production incident response.
- Commercial integration support.
- Custom feature development.
- One-on-one training.

## Remote Install

Remote Install is available through Ko-fi:

```text
https://ko-fi.com/nisbdev/commissions
```

Remote Install coordination email:

```text
install@nisb.me
```

Before ordering, prepare:

- VPS provider and OS version.
- Domain or subdomain, if any.
- Whether Docker is already installed.
- Preferred contact email.
- Target use case for NISB.
- Whether you want Room-as-MCP / LibreChat verification.

Do not send passwords, API keys, private SSH keys, bearer tokens, or production secrets through public issues or Ko-fi messages.

Secure access and installation details should be coordinated by email after the order.

## Security issues

Please do not open public GitHub issues for security vulnerabilities.

Report security issues to:

```text
security@nisb.me
```

Include enough detail to reproduce the issue safely, but do not include real secrets, production credentials, private user data, or private customer data unless a secure disclosure channel has been agreed.

See `SECURITY.md` for the security policy.

## Before opening an issue

Please include:

- NISB version or commit.
- Deployment method.
- Operating system.
- Docker version if applicable.
- Browser version if frontend-related.
- Clear reproduction steps.
- Expected behavior.
- Actual behavior.
- Relevant logs with secrets removed.
- Screenshots only if they do not contain private data.
- Whether the issue occurs on a fresh workspace or only with existing data.

---

## Response expectations

NISB is maintained independently.

Issue response time may vary.

For time-sensitive production help, private deployment, commercial integration, or enterprise support, contact the maintainer through the paid support or commercial licensing channels above.
